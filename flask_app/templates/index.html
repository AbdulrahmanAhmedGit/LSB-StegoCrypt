Project Overview

Understanding the design motivations and problem statements behind this implementation.

Core Purpose & Goal

Steganography conceals the existence of a message, while cryptography conceals its meaning. LSB-StegoCrypt integrates both paradigms into a unified command-line pipeline. This ensures that even if an attacker detects and extracts the hidden pixel payload, the data remains mathematically unreadable without the correct cryptographic key.

The Security Challenge

Standard LSB embedding is highly vulnerable to statistical detection (steganalysis) if the raw plaintext is embedded directly. Plaintext has high frequency patterns that stand out. By applying AES encryption first, we randomize the payload. Compressing the ciphertext then shrinks the footprint, reducing the number of modified pixels and maximizing stealth.

Step-by-Step Processing Flow

The sequence of operations used to embed and retrieve hidden text data.

01

Input Processing

The program accepts a raw secret text message and an authentication password from the user.

02

Fernet Encryption

Hashes the password with SHA-256 to derive a 256-bit key, encrypting the plaintext via AES-128 in CBC mode with HMAC-SHA256.

03

zlib Compression

Applies zlib compression to the ciphertext, minimizing the bitstream length to optimize pixel capacity.

04

LSB Pixel Injection

Injects the binary bits of the compressed data into the least significant bits of the image's RGB channels, appending a 16-bit stopper marker.

05

Extraction & Decryption

Scans pixel bytes, extracts the 0th bit of each color channel, halts at the 16-bit marker, decompresses, and decrypts the payload.

Security Layer Breakdown

The multi-layered protective envelope enclosing the hidden data.

Fernet (AES-128) Cryptography

Fernet guarantees that any message encrypted with it cannot be read or modified without the key. It uses AES in CBC mode with a 128-bit key for encryption, and HMAC-SHA256 for integrity verification.

Key Derivation Function

Instead of using plain text passwords as keys, the script processes the password using the SHA-256 hashing algorithm to derive a standard 32-byte url-safe base64 encoded key suitable for Fernet.

zlib Data Compression

Encrypted text is highly entropic and cannot be compressed. The script compresses the payload before bit-shifting. This reduces the number of altered pixels, which directly reduces statistical vulnerability.

LSB Steganography

By modifying only the least significant bit (bit 0) of the RGB channels, the pixel values change by at most 1 unit. This minor change is visually indistinguishable and preserves image structure.

Engineering Challenges & Solutions

Technical hurdles encountered during design and how they were overcome.

Constraint

Image Capacity Limits

Encrypted and compressed payloads can exceed the storage capacity of small images, causing out-of-bounds errors during bit injection.

Resolution

Calculates available bytes (Width * Height * 3 channels) and asserts that payload size is smaller than this value before processing, throwing a clean error if exceeded.

Constraint

End-of-Stream Detection

During decoding, the system needs to know exactly when to stop reading bits, otherwise it will extract noise bytes from unmodified pixels.

Resolution

Appends a 16-bit trailing delimiter ('0000000000000000') to the bitstream during encoding. The decoder scans pixels until this sequence is matched, discarding trailing noise.

Constraint

Password Verification

Decrypting with a wrong password causes the cryptography library to raise an exception, which crashes execution.

Resolution

Wraps decompression and decryption in a try-except structure. If an exception occurs, it intercepts the error and returns a clean 'Wrong Password' message to the user.

Constraint

Lossy Format Vulnerabilities

Formats like JPEG compress pixel values lossily, which destroys the embedded least significant bits during saving.

Resolution

Enforces saving strictly in lossless formats (PNG) and validates output extensions. The application automatically appends '.png' if it is missing.
